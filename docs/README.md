# CERT-FLOW documentation

Everything here is generated by a script and reproducible. Each results doc
names its `scripts/...` driver; the core sweep runs in ~100 s.

### Reading the result tables

Every metric column is marked, and methods are ranked **best → worst**:

> `↑` higher is better · `↓` lower is better · `·` informational ·
> **bold** = best in that column

For *coverage*, higher is safer but the real bar is **meeting or exceeding the
claimed/target level** — a valid certificate, not the largest number.
